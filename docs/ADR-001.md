# ADR-001: Use MCP Over Custom REST API

## Status
Accepted

## Context
We need a way to expose Databricks pipeline internal state, data quality metrics, and ML predictions to an AI agent. Historically, this might be done via a custom REST API or specialized LangChain tool wrappers.

## Decision
We will use the **Model Context Protocol (MCP)** to expose these capabilities.

## Rationale
- **Standardization:** MCP provides a standardized way for agents to discover and call tools. 
- **Native Support:** AgentBricks (Mosaic AI) is increasingly moving towards native MCP support.
- **Decoupling:** The MCP server can be redeployed or updated independently of the LLM or the underlying Spark pipeline.

## Consequences
- Requires the `mcp[server]` Python SDK.
- Team must learn the MCP protocol structure (JSON-RPC).
