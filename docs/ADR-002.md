# ADR-002: Single Supervisor Agent as Baseline

## Status
Accepted

## Context
The project requires a multi-agent orchestration pattern but is targeted at freshers.

## Decision
We will implement a **Single Supervisor Agent** that directly calls multiple tools, rather than a full multi-agent hierarchy.

## Rationale
- **Simplicity:** Multi-agent systems add significant complexity in state management, message routing, and debugging.
- **Evaluation:** It is easier to evaluate the "groundedness" and "tool selection" of a single agent compared to a distributed system.
- **Performance:** For the scope of this project (6-8 tools), a single context window is sufficient.

## Consequences
- The system prompt must be carefully tuned to manage tool routing.
- Advanced users can still extend this into the "Optional" multi-agent goal.
