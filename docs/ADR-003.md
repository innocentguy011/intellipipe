# ADR-003: scikit-learn Over Spark MLlib for ML Model

## Status
Accepted

## Context
We need to predict hourly data volume and detect anomalies.

## Decision
We will use **scikit-learn** for modeling instead of Spark MLlib.

## Rationale
- **Data Size:** The Gold layer aggregation results in ~2,160 rows for 90 days (1 row per hour). This easily fits in memory on a single node.
- **Algorithm Selection:** scikit-learn offers a wider variety of algorithms (Gradient Boosting Regressor, Isolation Forest) with simpler APIs compared to MLlib.
- **Iteration Speed:** Training is nearly instantaneous for this data size.

## Consequences
- The ML pipeline converts Spark DataFrames to Pandas for training.
- Not designed for row-level real-time streaming at million-event scale, but perfect for the hourly batch use case.
